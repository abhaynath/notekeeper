Git

Git Configuration

Setting username and email

  1. on Command Prompt, use following command 
        git config --global user.name = "Abhay Nath" 
    git config --global user.email = "abhaynath@gmail.com"
  2. To check if the user name and emailid successfully saved, 
        git config user.name
    git config user.email

Creating Repository

  1. Create an empty directory named 'myproject'. Run following command inside myproject 
    git init
  2. Run following command to check the status of the repository 
    git status
  3. Add 'index.html' new file in 'myproject' folder. To stage the index.html file, run following command 
    git add index.html
    To add all files, 
    git add --all
  4. To commit changes, run following command 
    git commit --message "initial commit for the project"
    or 
    git commit --m "initial commit for the project"

Delete Files

  1. Suppose we want to delete any file from the repository 
    git rm css/style.css
  2. This will delete the css folder and add a staged snapshot. To commit these changes, 
    git commit -m "Removed CSS folder"

Rename files

  1. If we want to rename style.css to screen.css 
    git mv css/style.css css/screen.css

Untrack files already added to git repository based on .gitignore

  1. Add the list of files/folder in gitignore files you want git to ignore.
  2. Commit the changes
  3. Remove everything from the repository. 
    git rm -r --cached .
    This command will remove everything from repository cached
  4. Re add everything 
    git add .
  5. Commit 
    git commit -m ".gitignore fix"

Branching

  1. Get list of all branches 
    git branch
  2. Create new branch 
    git branch new-homepage
  3. Switch to newly created branch, 
    git checkout new-homepage
  4. Check which branch is active currently, 
    git branch
  5. Merging Branch
    1. Suppose we have two branches (master & new-homepage). You have made few changes in 'new-home' branch and you want to merge these changes into master branch.
    2. Commit all the changes in 'new-homepage' branch.
    3. Switch to 'master' branch.
    4. git merge new-homepage
      This will merge all the changes from 'new-homepage' to 'master'
  6. Delete a local branch 
    git branch -d branch_name

Remote Repository

  1. To clone the repository on GitHub, copy the path of project
  2. Create an empty folder. Run following command from the directory, 
    git clone https://github.com/abhaynath/snake.git
    This will create a repository called 'origin' 
    git push origin master
  3. Get list of branches on remote 
    git remote show origin
  4. Delete a remote branch 
    git push origin --delete branch_name

Stashing

  1. If you do not want to commit some unfinished files changes, but you may need them in future, you need to stash those files. 
    git stash
  2. To get the list of stash 
    git stash list
      Output : 
  stash@{0}: WIP on style: af6811f merged changed
  stash@{1}: WIP on style: af6811f merged changed
  stash@{2}: WIP on style: af6811f merged changed
  3. To apply stash, 
    git stash apply stash@{1}

Commands

  1. Get list of changes in file 
    git diff <filename>
  2. Get list of Commits 
    git reflog <filename>
    git reflog
  3. Compare the changes between two files, 
    git diff <id1> <id2>
  4. Replaced master branch with another branch code completely 
      git checkout new_branch
  git merge -s ours master
  git checkout master
  git merge new_branch

Rollback Changes

  1. To revert uncommited changes in local repository
  2. Method 1 : For individual files 
    git checkout <filename>
    for all files since last commit 
    git reset --hard
  3. Method 2 : 
    git checkout .
